---
title: Redact API
description: Remove sensitive information (PII/PHI) from text using the Redact API
---

The Redact API removes sensitive information (PII/PHI) from text, replacing it with safe placeholders.

## Base URL

```text title="Base URL"
https://app.superagent.sh/api
```

## Authentication

All API requests require authentication using your Superagent API key. Include it in the request headers:

```http title="Authentication Headers"
Authorization: Bearer YOUR_API_KEY
x-superagent-api-key: YOUR_API_KEY
```

## Endpoint

```text title="Endpoint"
POST /redact
```

## Request

**Headers:**
```http title="Request Headers"
Content-Type: application/json
Authorization: Bearer YOUR_API_KEY
x-superagent-api-key: YOUR_API_KEY
```

**Body:**
```json title="Request Body"
{
  "prompt": "string",           // Required: The text to redact
  "entities": ["string", ...]   // Optional: Custom entities to redact
}
```

## Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `prompt` | string | Yes | The text containing sensitive information to redact |
| `entities` | array of strings | No | Natural language descriptions of custom entities to redact (e.g., "credit card numbers", "email addresses") |

## Response

**Success (200 OK):**
```json title="Response Example"
{
  "id": "dc28f957-f70c-410d-9b2b-37fc808f23e9",
  "model": "superagent-lm-redact-20b",
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "My email is <EMAIL_REDACTED> and SSN is 123-45-6789",
        "reasoning": "We need to redact email only. So replace john@example.com with <EMAIL_REDACTED>. Keep SSN unchanged. Output: \"My email is <EMAIL_REDACTED> and SSN is 123-45-6789\"."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 333,
    "completion_tokens": 81,
    "total_tokens": 414
  }
}
```

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier for the request |
| `model` | string | The redaction model used |
| `choices[].message.content` | string | The text with sensitive information redacted |
| `choices[].message.reasoning` | string | Explanation of what was redacted and why |
| `choices[].finish_reason` | string | Reason the generation stopped (typically "stop") |
| `usage` | object | Token usage statistics for the request |

## Detected PII/PHI Types

The redaction API automatically detects and redacts:

| Type | Replacement Token | Example |
|------|------------------|---------|
| Email addresses | `<EMAIL_REDACTED>` | john@example.com → `<EMAIL_REDACTED>` |
| Social Security Numbers | `<SSN_REDACTED>` | 123-45-6789 → `<SSN_REDACTED>` |
| Credit cards | `<CC_REDACTED>` | 4532-1234-5678-9010 → `<CC_REDACTED>` |
| Phone numbers | `<PHONE_REDACTED>` | (555) 123-4567 → `<PHONE_REDACTED>` |
| IP addresses | `<IP_REDACTED>` | 192.168.1.1 → `<IP_REDACTED>` |
| API keys & tokens | `<API_KEY_REDACTED>` | sk-1234567890abcdef → `<API_KEY_REDACTED>` |
| AWS access keys | `<AWS_KEY_REDACTED>` | AKIAIOSFODNN7EXAMPLE → `<AWS_KEY_REDACTED>` |
| Bearer tokens | `Bearer <TOKEN_REDACTED>` | Bearer eyJhbGc... → `Bearer <TOKEN_REDACTED>` |
| MAC addresses | `<MAC_REDACTED>` | 00:1B:44:11:3A:B7 → `<MAC_REDACTED>` |
| Medical record numbers | `<MRN_REDACTED>` | MRN-123456 → `<MRN_REDACTED>` |
| Passport numbers | `<PASSPORT_REDACTED>` | A12345678 → `<PASSPORT_REDACTED>` |
| IBAN | `<IBAN_REDACTED>` | GB29NWBK60161331926819 → `<IBAN_REDACTED>` |
| ZIP codes | `<ZIP_REDACTED>` | 90210 → `<ZIP_REDACTED>` |

## Examples

### Basic Redaction

**Request:**
```bash title="cURL Request"
curl -X POST https://app.superagent.sh/api/redact \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "x-superagent-api-key: YOUR_API_KEY" \
  -d '{
    "prompt": "My email is john@example.com and my SSN is 123-45-6789"
  }'
```

**Response:**
```json title="Response"
{
  "id": "xyz789",
  "model": "superagent-lm-redact-20b",
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "My email is <EMAIL_REDACTED> and my SSN is <SSN_REDACTED>",
        "reasoning": "Redacted email address and Social Security Number for privacy protection."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 250,
    "completion_tokens": 65,
    "total_tokens": 315
  }
}
```

### Custom Entity Redaction

**Request:**
```bash title="cURL Request"
curl -X POST https://app.superagent.sh/api/redact \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "x-superagent-api-key: YOUR_API_KEY" \
  -d '{
    "prompt": "Call me at 555-1234 or email support@company.com",
    "entities": ["phone numbers", "email addresses"]
  }'
```

**Response:**
```json title="Response"
{
  "id": "custom123",
  "model": "superagent-lm-redact-20b",
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "Call me at <PHONE_REDACTED> or email <EMAIL_REDACTED>",
        "reasoning": "Redacted phone number and email address as specified in entities list."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 280,
    "completion_tokens": 70,
    "total_tokens": 350
  }
}
```
